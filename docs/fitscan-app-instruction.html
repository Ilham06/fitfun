📋
Phase 2 — Onboarding Wizard
4 steps · back/next navigation · progress indicator · blocked until complete
1
Profile — /onboarding/profile
Name (pre-filled from Google account), age, gender, current weight (kg/lbs toggle), height (cm/ft toggle).
2
Program — /onboarding/program
3 pill cards: Bulking (+15% cal · build mass), Cutting (−20% cal · lose fat), Maintenance (= TDEE · stay steady). Only one selectable.
3
Activity Level — /onboarding/activity
4 options with multipliers: Sedentary ×1.2 · Lightly Active ×1.375 · Moderately Active ×1.55 · Very Active ×1.725. Used in Mifflin-St Jeor TDEE formula.
4
Goal & Review — /onboarding/goal
Optional target weight. Auto-calculated TDEE, daily cal target, macro split (P 30% / C 45% / F 25%) shown live. Confirm → POST /api/profile → set onboardingDone = true → redirect /dashboard.
Mifflin-St Jeor BMR TDEE × program modifier
authenticated shell
🏠
Phase 3 — Main App (4-tab Shell)
Bottom tab nav · Home · Scan (FAB) · Progress · Profile
🏠
Home / Dashboard — /dashboard
Macro ring summary (4 rings), today's meal log grouped by type (B/L/D/Snack), water tracker, streak counter, inline AI daily tip card (GPT-4o, generated once per day, cached).
📷
Scan — /scan
Central FAB. Opens scan screen with mode toggle: Food (camera / gallery / barcode) or Body (camera / gallery for measurement photo). Both modes use GPT-4o Vision.
📊
Progress — /progress
Weight chart with goal overlay, per-metric body measurement charts, macro adherence bars (weekly), 30-day calorie trend, composition delta summary cards.
👤
Profile — /profile
Edit profile & recalculate TDEE, change program, notification preferences, export data as CSV, sign out.
feature flows
🍎
Feature Flow A — Food Scan & Log
All recognition via GPT-4o Vision · barcode fallback to Open Food Facts
1
Open Scan → Food Mode
Camera viewfinder opens. Three entry paths: Live camera snap, Gallery import, or "Search food manually" text input fallback.
2a
Image Capture → GPT-4o Vision
Photo (any food or packaged label) → POST /api/food/scan → image sent to OpenAI with system prompt → returns JSON: { name, portionG, kcal, proteinG, carbG, fatG, fiberG, sodiumMg, confidence }
GPT-4o Vision (gpt-4o)
2b
Barcode Detected (ZXing)
ZXing auto-detects EAN-13/UPC-A in viewfinder frame → POST /api/food/barcode → query Open Food Facts API → if found, return structured nutrition. If not found, fall back to GPT-4o Vision with the product photo.
ZXing + Open Food FactsGPT-4o fallback
3
Nutrition Result Card — /scan/result
Food name (editable), confidence badge, full macro breakdown, portion slider (0.5× – 3×, all values scale live), meal tag picker (Breakfast / Lunch / Dinner / Snack).
4
Save → Dashboard Update
POST /api/food/log → Prisma creates FoodLog → Dashboard macro rings update optimistically → success toast. User returns to Dashboard or stays on Scan to add another item.
📏
Feature Flow B — Body Measurement Scan
Photo → GPT-4o Vision extracts + maps measurement fields
1
Open Scan → Body Mode
Camera or gallery picker. On-screen overlay guide for framing measurement tape, scale display, or handwritten note for best accuracy.
2
Image → GPT-4o Vision Extraction
Photo → POST /api/body/scan → sent to OpenAI GPT-4o with structured prompt → returns JSON mapping: { weightKg, waistCm, chestCm, hipsCm, armsCm, thighsCm, neckCm, bodyFatPct }. Unknown fields returned as null.
GPT-4o Vision (gpt-4o)Structured JSON output
3
Confirmation Screen — /scan/body-confirm
Each extracted value shown in editable field. Null fields left blank for manual fill. BMI auto-calculated live from weight + stored height. Delta vs. previous entry shown in green/red.
4
Save → Progress Charts Update
POST /api/body/log → Prisma creates BodyMeasurement with timestamp → Progress tab charts update immediately. Success toast confirms save.

API Route Map

Authentication (NextAuth)
GET/api/auth/[...nextauth]Session check, Google OAuth callback
POST/api/auth/[...nextauth]Sign in / sign out
Onboarding & Profile
GET/api/profileGet current user profile + daily targetsauth
POST/api/profileCreate profile, calc TDEE + macros, set onboardingDoneauth
PATCH/api/profileUpdate weight / program / activity, recalculate targetsauth
Food Scan & Logging
POST/api/food/scanImage → GPT-4o Vision → JSON nutrition resultauth
POST/api/food/barcodeBarcode string → Open Food Facts → nutrition (GPT-4o fallback)auth
GET/api/food/searchText search nutritional database (?q=)auth
GET/api/food/logGet food logs (?date=YYYY-MM-DD)auth
POST/api/food/logCreate FoodLog from confirmed nutrition resultauth
PATCH/api/food/log/[id]Edit portion or meal type of a logged entryauth
DEL/api/food/log/[id]Delete a food log entryauth
Body Measurement Scan
POST/api/body/scanImage → GPT-4o Vision → extracted measurement JSONauth
GET/api/body/logGet measurement history (?limit=30)auth
POST/api/body/logSave confirmed body measurements, auto-calc BMIauth
DEL/api/body/log/[id]Delete a measurement recordauth
AI Features (GPT-4o)
POST/api/ai/daily-tipGenerate daily tip + meal suggestions (cached once/day per user)auth
POST/api/ai/weekly-reportGenerate 7-day insight report (cached once/week)auth
Dashboard & Progress
GET/api/dashboard/todayToday's calorie, macro, water totals + streak countauth
GET/api/progress/nutritionMacro adherence data for charts (?days=30)auth
GET/api/progress/bodyAll body measurement history for chart renderingauth
GET/api/export/csvDownload all user data as CSVauth
